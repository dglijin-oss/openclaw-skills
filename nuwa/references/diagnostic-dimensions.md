# 需求诊断维度表

用于 Phase 0B：从用户模糊需求反推最合适的蒸馏对象。

| 需求维度 | 典型表达 | 思维框架方向 | 推荐人物参考 |
|---------|---------|------------|------------|
| 决策与判断 | 「怎么做更好的决策」「总是选错」「分析瘫痪」 | 多元思维模型、逆向思考、概率思维 | 芒格、塔勒布 |
| 表达与写作 | 「想把复杂的事说清楚」「文章没人看」「写得无聊」 | 费曼式简化、故事化思维、类比能力 | 费曼 |
| 创业与商业 | 「想做独立开发」「商业模式想不通」「找不到PMF」 | 第一性原理、杠杆思维、产品克制 | 马斯克、雷军 |
| 教学与传播 | 「讲课没人听」「学生理解不了」「知识传递效率低」 | 从已知到未知、隐喻教学、最少必要知识 | 费曼 |
| 批判思维 | 「总被忽悠」「想识别不靠谱的说法」「看不透本质」 | 证伪思维、演化论视角、认知偏差识别 | 芒格 |
| 内容创作 | 「做视频没流量」「不知道拍什么」「内容没特色」 | 注意力工程、测试迭代、受众心理 | MrBeast |
| 人生策略 | 「职业方向迷茫」「时间总不够」「焦虑」 | 长期主义、杠杆选择、复利思维 | 巴菲特 |
| 风险与不确定性 | 「怎么应对黑天鹅」「投资总亏」「太保守/太冒险」 | 反脆弱、凸性策略、尾部风险管理 | 塔勒布 |
| 设计与产品 | 「用户体验差」「产品没特色」「不知道做减法」 | 极简主义、用户心理模型、约束即创意 | 乔布斯 |
| 幽默与表达力 | 「说话没意思」「想让内容更有趣」「太严肃了」 | 荒诞对比、预期违背、自嘲式权威 | - |

## 候选推荐格式

```
### 候选1: [人名/主题]  ⚡已有Skill / 🆕需要蒸馏

**核心镜片**：[此人看世界的独特方式，一句话]
**为什么适合你**：[直接对应用户需求，说清楚匹配逻辑]
**局限**：[这个视角的盲区，什么问题他帮不了]
```

### 推荐原则
- 不超过3个候选
- 已有Skill优先展示（即插即用）
- 候选之间要有差异性
- 必须说清楚局限
- 推荐要具体到「这个人的哪个思维模型」匹配需求
